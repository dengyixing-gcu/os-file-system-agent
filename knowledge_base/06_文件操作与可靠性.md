## 一、基本文件操作

### 2. 创建文件（create）

**功能**：在文件系统中创建一个新文件。

**操作步骤**：
1. 在目录中分配一个目录项
2. 填写文件名
3. 分配 FCB 或 inode
4. 初始化文件属性（大小=0，权限等）
5. 返回文件描述符

```c
// Unix 系统调用
int create(const char *pathname, mode_t mode);

// 示例
int fd = create("/home/alice/test.txt", 0644);
// 返回文件描述符 fd，失败返回 -1

// C 标准库函数
FILE *fp = fopen("/home/alice/test.txt", "w");
```

**创建时的属性设置**：
```
┌─────────────────────────────────────────────────┐
│           创建文件时的属性                       │
├─────────────────────────────────────────────────┤
│  文件名：test.txt                              │
│  文件类型：普通文件                            │
│  文件大小：0 字节                              │
│  所有者：当前用户                              │
│  权限：0644 (rw-r--r--)                        │
│  创建时间：当前时间                            │
│  物理地址：暂未分配（写入时分配）              │
└─────────────────────────────────────────────────┘
```

### 3. 删除文件（delete/unlink）

**功能**：从文件系统中删除一个文件。

**操作步骤**：
1. 检查文件是否存在
2. 检查用户是否有删除权限
3. 检查文件是否被打开（引用计数）
4. 从目录中删除目录项
5. 回收文件占用的磁盘空间
6. 释放 FCB 或 inode

```c
// Unix 系统调用
int unlink(const char *pathname);
int remove(const char *pathname);

// 示例
unlink("/home/alice/old.txt");  // 删除文件

// 强制删除（忽略权限）
rm -f /home/alice/old.txt
```

**注意事项**：
- 文件被打开时，Unix 系统允许删除（引用计数为 0 时才真正删除）
- Windows 系统不允许删除已打开的文件

### 4. 打开文件（open）

**功能**：建立用户与文件之间的连接，为后续操作做准备。

**为什么需要打开文件？**
- 避免每次操作都查找目录
- 将 FCB/inode 读入内存
- 分配系统资源（缓冲区、文件描述符）

```c
// Unix 系统调用
int open(const char *pathname, int flags, mode_t mode);

// flags 参数
O_RDONLY  // 只读
O_WRONLY  // 只写
O_RDWR    // 读写
O_CREAT   // 文件不存在则创建
O_APPEND  // 追加模式
O_TRUNC   // 清空文件

// 示例
int fd = open("/home/alice/data.txt", O_RDWR);
if (fd == -1) {
    perror("open failed");
}
```

**打开文件后的系统资源**：

```
┌─────────────────────────────────────────────────┐
│          打开文件后的系统状态                    │
│                                                 │
│  用户进程                      操作系统          │
│  ┌─────────┐                 ┌─────────────┐   │
│  │  进程   │                 │ 系统打开    │   │
│  │         │                 │ 文件表      │   │
│  │ 文件    │────────────────→│ ┌─────────┐ │   │
│  │ 描述符  │  文件描述符表   │ │文件 1   │ │   │
│  │   3     │                 │ │inode 指针│ │   │
│  │         │                 │ │读写位置 │ │   │
│  └─────────┘                 │ │打开计数 │ │   │
│                              │ └─────────┘ │   │
│                              └─────────────┘   │
│                                     │          │
│                                     ↓          │
│                              ┌─────────────┐   │
│                              │    inode    │   │
│                              │ (内存副本)  │   │
│                              └─────────────┘   │
└─────────────────────────────────────────────────┘
```

### 5. 关闭文件（close）

**功能**：断开用户与文件的连接，释放系统资源。

**操作步骤**：
1. 将缓冲区数据写入磁盘（如果有未写入的数据）
2. 更新文件属性（大小、修改时间）
3. 减少打开文件计数
4. 释放文件描述符
5. 如果计数为 0，释放内存中的 inode

```c
// Unix 系统调用
int close(int fd);

// 示例
close(fd);

// C 标准库
fclose(fp);
```

**为什么需要关闭文件？**
- 释放有限的文件描述符资源
- 确保数据写入磁盘（避免数据丢失）
- 更新文件元数据

### 6. 读文件（read）

**功能**：从文件中读取数据到内存缓冲区。

```c
// Unix 系统调用
ssize_t read(int fd, void *buf, size_t count);

// 参数说明
// fd    - 文件描述符
// buf   - 读取数据的目的地缓冲区
// count - 最多读取的字节数

// 返回值
// > 0   - 实际读取的字节数
// = 0   - 到达文件末尾 (EOF)
// -1    - 出错

// 示例
char buffer[1024];
ssize_t n = read(fd, buffer, sizeof(buffer));
if (n > 0) {
    // 处理读取的 n 个字节
} else if (n == 0) {
    // 文件结束
} else {
    // 出错处理
}
```

**读取过程**：

```
┌─────────────────────────────────────────────────┐
│              读文件流程                          │
│                                                 │
│  1. 检查文件描述符有效性                        │
│  2. 检查读权限                                  │
│  3. 从当前读写位置开始读取                      │
│  4. 如果数据不在内存，从磁盘读入缓冲区          │
│  5. 将数据从内核缓冲区复制到用户缓冲区          │
│  6. 更新读写位置                                │
│  7. 返回实际读取的字节数                        │
│                                                 │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐     │
│  │ 磁盘    │───→│ 内核    │───→│ 用户    │     │
│  │ 文件    │    │ 缓冲区  │    │ 缓冲区  │     │
│  └─────────┘    └─────────┘    └─────────┘     │
└─────────────────────────────────────────────────┘
```

### 7. 写文件（write）

**功能**：将内存数据写入文件。

```c
// Unix 系统调用
ssize_t write(int fd, const void *buf, size_t count);

// 参数说明
// fd    - 文件描述符
// buf   - 要写入数据的源缓冲区
// count - 要写入的字节数

// 返回值
// > 0   - 实际写入的字节数
// -1    - 出错

// 示例
const char *msg = "Hello, World!";
ssize_t n = write(fd, msg, strlen(msg));
if (n == -1) {
    perror("write failed");
}
```

**写入过程**：

```
┌─────────────────────────────────────────────────┐
│              写文件流程                          │
│                                                 │
│  1. 检查文件描述符有效性                        │
│  2. 检查写权限                                  │
│  3. 从当前读写位置开始写入                      │
│  4. 将数据从用户缓冲区复制到内核缓冲区          │
│  5. 必要时分配新的磁盘块                        │
│  6. 异步将缓冲区数据写入磁盘                    │
│  7. 更新读写位置和文件大小                      │
│  8. 返回实际写入的字节数                        │
│                                                 │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐     │
│  │ 用户    │───→│ 内核    │───→│ 磁盘    │     │
│  │ 缓冲区  │    │ 缓冲区  │    │ 文件    │     │
│  └─────────┘    └─────────┘    └─────────┘     │
│                          (异步写入)             │
└─────────────────────────────────────────────────┘
```

### 8. 文件定位（seek/lseek）

**功能**：移动文件的读写位置指针。

```c
// Unix 系统调用
off_t lseek(int fd, off_t offset, int whence);

// whence 参数
SEEK_SET  // 从文件开头开始计算
SEEK_CUR  // 从当前位置开始计算
SEEK_END  // 从文件末尾开始计算

// 示例
// 移动到文件开头
lseek(fd, 0, SEEK_SET);

// 向前移动 100 字节
lseek(fd, 100, SEEK_CUR);

// 移动到文件末尾
lseek(fd, 0, SEEK_END);

// 获取当前读写位置
off_t pos = lseek(fd, 0, SEEK_CUR);
```

**定位示意图**：

```
┌─────────────────────────────────────────────────┐
│           文件定位示意图                         │
│                                                 │
│  文件内容：                                      │
│  0    100    200    300    400    500          │
│  │────│────│────│────│────│────│               │
│  ↑                    ↑         ↑               │
│  │                    │         │               │
│ SEEK_SET(0)       SEEK_CUR   SEEK_END          │
│ 文件开头          当前位置    文件末尾          │
│                                                 │
│  lseek(fd, 0, SEEK_SET)   → 位置=0             │
│  lseek(fd, 50, SEEK_CUR)  → 位置=当前位置 +50  │
│  lseek(fd, 0, SEEK_END)   → 位置=文件大小      │
└─────────────────────────────────────────────────┘
```

### 9. 获取文件属性（stat/fstat）

**功能**：获取文件的元数据信息。

```c
// Unix 系统调用
#include <sys/stat.h>

int stat(const char *pathname, struct stat *buf);
int fstat(int fd, struct stat *buf);

// stat 结构
struct stat {
    mode_t     st_mode;     // 文件类型和权限
    ino_t      st_ino;      // inode 号
    uid_t      st_uid;      // 所有者 UID
    gid_t      st_gid;      // 组 ID
    off_t      st_size;     // 文件大小（字节）
    time_t     st_atime;    // 最后访问时间
    time_t     st_mtime;    // 最后修改时间
    time_t     st_ctime;    // 状态改变时间
    // ... 其他字段
};

// 示例
struct stat sb;
if (stat("/home/alice/file.txt", &sb) == 0) {
    printf("文件大小：%ld 字节\n", sb.st_size);
    printf("inode 号：%ld\n", sb.st_ino);
}
```

## 三、文件备份

### 1. 备份的目的

- 防止数据丢失（硬件故障、人为错误）
- 灾难恢复
- 历史版本保留
- 合规要求

### 2. 备份类型

```
┌─────────────────────────────────────────────────┐
│              备份类型对比                        │
├─────────────────────────────────────────────────┤
│                                                 │
│  1. 完全备份 (Full Backup)                      │
│     ┌─────────────────────────────────────┐    │
│     │ 备份所有选定的文件                   │    │
│     │ 优点：恢复简单，只需一份备份         │    │
│     │ 缺点：时间长，占用空间大             │    │
│     └─────────────────────────────────────┘    │
│                                                 │
│  2. 增量备份 (Incremental Backup)               │
│     ┌─────────────────────────────────────┐    │
│     │ 只备份上次备份后变化的文件           │    │
│     │ 优点：时间短，占用空间小             │    │
│     │ 缺点：恢复需要多份备份               │    │
│     └─────────────────────────────────────┘    │
│                                                 │
│  3. 差异备份 (Differential Backup)              │
│     ┌─────────────────────────────────────┐    │
│     │ 备份上次完全备份后变化的文件         │    │
│     │ 优点：恢复只需两份备份               │    │
│     │ 缺点：备份时间逐渐增加               │    │
│     └─────────────────────────────────────┘    │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 3. 备份策略示例

```

│                                                 │

│  周二：增量备份（相对于周一）                   │
│  周三：增量备份（相对于周二）                   │
```

### 2. 日志文件系统

**原理**：在进行实际修改前，先将操作记录到日志中。

```
┌─────────────────────────────────────────────────┐
│          日志文件系统工作原理                    │
│                                                 │
│  正常操作流程：                                  │
│  1. 写入日志：记录将要进行的操作                │
│  2. 提交日志：标记日志已提交                    │
│  3. 执行操作：实际修改文件系统                  │
│  4. 完成日志：标记操作已完成                    │
│                                                 │
│  系统崩溃恢复：                                  │
│  1. 检查日志，找到未完成的操作                  │
│  2. 重做已提交但未完成的操作                    │
│  3. 撤销未完成的操作                            │
│  4. 文件系统恢复到一致状态                      │
│                                                 │
│  典型系统：ext3/ext4, NTFS, XFS, ZFS           │
└─────────────────────────────────────────────────┘
```

### 3. 快照（Snapshot）

**原理**：在某一时刻创建文件系统的"照片"。

```
┌─────────────────────────────────────────────────┐
│              快照机制示意图                      │
│                                                 │
│  时间点 T0: 创建快照 S0                         │
│  ┌─────────────────────────────────────────┐    │
│  │ 文件系统状态：A B C D E                 │    │
│  │ 快照 S0: [A B C D E]                    │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
│  时间点 T1: 修改文件 C → C'                     │
│  ┌─────────────────────────────────────────┐    │
│  │ 文件系统状态：A B C' D E                │    │
│  │ 快照 S0: [A B C D E]  ← 保留原数据 C    │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
│  恢复：从快照 S0 恢复，文件系统回到 T0 状态       │
│                                                 │
│  技术：写时复制（Copy-on-Write）               │
└─────────────────────────────────────────────────┘
```

### 4. RAID 技术

**原理**：多磁盘冗余存储，提高可靠性。

| RAID 级别 | 描述 | 冗余 | 利用率 |
|-----------|------|------|--------|
| RAID 0 | 条带化 | 无 | 100% |
| RAID 1 | 镜像 | 完整备份 | 50% |
| RAID 5 | 条带化 + 分布式奇偶校验 | 单盘冗余 | (n-1)/n |
| RAID 6 | 条带化 + 双重奇偶校验 | 双盘冗余 | (n-2)/n |
| RAID 10 | 镜像 + 条带化 | 完整备份 | 50% |

## 五、典型例题

**例题 1**：简述 open() 和 close() 系统调用的作用。

**答案**：
- **open()**：
  1. 根据文件名查找目录，找到 FCB/inode
  2. 将 FCB/inode 读入内存
  3. 在系统打开文件表中创建表项
  4. 在进程文件描述符表中分配描述符
  5. 返回文件描述符供后续操作使用

- **close()**：
  1. 将缓冲区数据写入磁盘
  2. 更新文件属性（大小、时间戳）
  3. 减少打开文件计数
  4. 释放文件描述符和系统资源

**例题 2**：比较完全备份、增量备份和差异备份的优缺点。

**答案**：
| 备份类型 | 优点 | 缺点 |
|----------|------|------|
| 完全备份 | 恢复简单 | 时间长、空间大 |
| 增量备份 | 时间短、空间小 | 恢复需要多份备份 |
| 差异备份 | 恢复只需两份备份 | 备份时间逐渐增加 |

**例题 3**：为什么日志文件系统可以提高可靠性？

**答案**：
1. 所有修改操作先记录到日志
2. 系统崩溃后，可通过日志恢复
3. 重做已提交但未完成的操作
4. 撤销未完成的操作
5. 确保文件系统始终处于一致状态
6. 避免数据损坏和丢失
