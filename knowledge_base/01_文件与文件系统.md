# 6.1 文件与文件系统

## 一、文件的定义

**文件（File）** 是具有文件名的一组相关信息的集合，是操作系统中数据存储的基本单位。

### 核心特征：
- **逻辑完整性**：文件是用户视角下完整的数据单元
- **持久性**：文件存储在外部存储器（如硬盘）上，断电后数据不丢失
- **命名性**：每个文件都有唯一的文件名供用户识别和访问
- **抽象性**：文件屏蔽了底层存储设备的物理细节

> 📌 **通俗理解**：文件就像电脑里的"文档"，你可以把它想象成现实生活中的文件夹里的纸张，操作系统帮你管理这些纸张，你只需要知道文件名就能找到它。

## 二、文件类型

### 1. 按用途分类

| 类型 | 说明 | 示例 |
|------|------|------|
| **系统文件** | 操作系统运行所需的文件 | 内核文件、驱动程序 |
| **用户文件** | 用户创建和使用的文件 | 文档、图片、程序 |
| **库文件** | 供程序调用的共享代码 | .dll、.so、.lib 文件 |

### 2. 按数据形式分类

| 类型 | 说明 | 示例 |
|------|------|------|
| **源文件** | 源代码文件 | .c、.java、.py |
| **目标文件** | 编译后的机器代码 | .o、.obj |
| **可执行文件** | 可直接运行的程序 | .exe、.out |
| **文本文件** | 字符数据 | .txt、.md |
| **二进制文件** | 二进制数据 | .jpg、.mp4 |

### 3. 按存取控制属性分类

| 类型 | 说明 |
|------|------|
| **只读文件** | 只能读取，不能修改 |
| **读写文件** | 可读取也可修改 |
| **可执行文件** | 可作为程序运行 |
| **不保护文件** | 无访问限制 |

## 三、文件属性

文件属性是描述文件特征信息的元数据，通常存储在文件控制块（FCB）或索引节点（inode）中。

### 常见文件属性：

```
┌─────────────────────────────────────────┐
│              文件属性                    │
├─────────────────────────────────────────┤
│  文件名        →  文件的标识符号         │
│  文件类型      →  普通文件/目录/链接等   │
│  文件大小      →  文件占用的字节数       │
│  文件位置      →  在存储设备上的物理地址 │
│  创建时间      →  文件创建的时间戳       │
│  最后访问时间  →  最后一次读取的时间     │
│  最后修改时间  →  最后一次写入的时间     │
│  所有者        →  文件创建者的用户 ID    │
│  访问权限      →  读/写/执行权限设置     │
└─────────────────────────────────────────┘
```

### 📝 举例说明

以 Linux 系统为例，使用 `ls -l` 命令查看文件属性：

```
-rw-r--r-- 1 user group  4096  Mar 26 10:00  document.txt
││││││││││ │ │    │      │     │            │
││││││││││ │ │    │      │     │            └─ 文件名
││││││││││ │ │    │      │     └─ 最后修改时间
││││││││││ │ │    │      └─ 文件大小 (字节)
││││││││││ │ │    └─ 所属组
││││││││││ │ └─ 所有者
││││││││││ └─ 硬链接数
│││││││││└─ 组权限 (r-- 表示只读)
││││││││└─ 其他用户权限 (r-- 表示只读)
│││││││└─ 所有者权限 (rw- 表示可读写)
││││││└─ 特殊权限位
│││││└─ 文件类型 (- 表示普通文件，d 表示目录)
```

## 四、文件系统的功能

**文件系统（File System）** 是操作系统中负责管理和存取文件信息的软件机构。

### 核心功能：

```
┌────────────────────────────────────────────────────┐
│                  文件系统功能                       │
├────────────────────────────────────────────────────┤
│                                                    │
│  1️⃣ 文件管理                                        │
│     • 文件逻辑结构到物理结构的映射                  │
│     • 文件存储空间管理                            │
│     • 文件目录管理                                │
│                                                    │
│  2️⃣ 目录管理                                       │
│     • 目录结构维护                                │
│     • 路径名解析                                  │
│     • 快速定位文件                                │
│                                                    │
│  3️⃣ 文件存储空间管理                               │
│     • 磁盘空间分配与回收                          │
│     • 空闲空间管理                                │
│     • 磁盘块分配算法                              │
│                                                    │
│  4️⃣ 文件共享与保护                                 │
│     • 多用户文件共享机制                          │
│     • 访问权限控制                                │
│     • 文件加密与认证                              │
│                                                    │
│  5️⃣ 文件操作接口                                   │
│     • 系统调用接口 (open/read/write/close)        │
│     • 命令行接口                                  │
│     • 图形界面接口                                │
│                                                    │
└────────────────────────────────────────────────────┘
```

## 五、文件系统的层次结构

文件系统采用分层设计，自顶向下分为多个层次：

```
┌─────────────────────────────────────────────────┐
│            用户应用程序层                         │
│         (User Applications)                     │
├─────────────────────────────────────────────────┤
│            逻辑文件系统层                         │
│       (Logical File System)                     │
│   • 处理符号文件名、路径名解析                    │
│   • 维护打开文件表                               │
│   • 提供文件操作 API                            │
├─────────────────────────────────────────────────┤
│            文件组织层                             │
│         (File Organization)                     │
│   • 逻辑块到物理块的映射                         │
│   • 文件分配表管理                              │
│   • 空闲空间管理                                │
├─────────────────────────────────────────────────┤
│            基本文件系统层                         │
│        (Basic File System)                      │
│   • 发出磁盘 I/O 命令                            │
│   • 磁盘块读写操作                              │
│   • 设备驱动程序接口                            │
├─────────────────────────────────────────────────┤
│            I/O 控制层                             │
│           (I/O Control)                         │
│   • 设备控制器                                  │
│   • 中断处理                                    │
│   • DMA 传输                                    │
├─────────────────────────────────────────────────┤
│            物理设备层                             │
│        (Physical Device)                        │
│   • 磁盘/SSD 等存储介质                          │
│   • 磁道、扇区、柱面                            │
└─────────────────────────────────────────────────┘
```

### 各层次职责说明：

| 层次 | 职责 | 关键数据结构 |
|------|------|-------------|
| **逻辑文件系统** | 解析文件名、权限检查、打开文件管理 | 打开文件表、目录缓存 |
| **文件组织层** | 逻辑块号→物理块号转换、空间分配 | 文件分配表、inode |
| **基本文件系统** | 磁盘块读写、缓冲区管理 | 磁盘缓冲区 |
| **I/O 控制层** | 设备调度、中断处理、DMA 控制 | 设备控制块、I/O 请求队列 |
| **物理设备** | 实际数据存储 | 磁盘扇区 |

## 六、易混淆点对比

### 🔄 文件 vs 文件系统

| 对比项 | 文件 | 文件系统 |
|--------|------|----------|
| **定义** | 数据的逻辑集合 | 管理文件的软件机构 |
| **角色** | 被管理的对象 | 管理者 |
| **存在形式** | 具体的数据内容 | 操作系统的一部分 |
| **类比** | 图书馆里的书 | 图书馆的管理系统 |

### 🔄 逻辑结构 vs 物理结构

| 对比项 | 逻辑结构 | 物理结构 |
|--------|----------|----------|
| **视角** | 用户视角 | 系统视角 |
| **关注点** | 数据如何组织 | 数据如何存储 |
| **独立性** | 用户可见 | 对用户透明 |
| **例子** | 记录式文件 | 连续分配、链接分配 |

## 七、典型例题

**例题 1**：文件系统中，文件控制块（FCB）的主要作用是？

**答案**：存储文件的属性信息，是文件存在的唯一标志，操作系统通过 FCB 来管理和控制文件。

**例题 2**：简述文件系统的层次结构中，逻辑文件系统层和文件组织层的区别。

**答案**：
- 逻辑文件系统层：处理符号文件名，进行路径名解析，维护打开文件表，提供用户接口
- 文件组织层：负责逻辑块号到物理块号的映射，管理文件存储空间分配
